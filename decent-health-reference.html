Database Schema

weight_logs

One row per day of weight entry.

ColumnTypeDescription
id PKuuidPrimary key, auto-generated
log_datedateDate of weight entry — UNIQUE
weightnumeric(5,1)Raw weight in lbs (e.g. 238.5)
created_attimestamptzAuto-set on insert

weight_context

Optional daily food/behavior score — captured the morning after (context_date = yesterday).

ColumnTypeDescription
id PKuuidPrimary key
context_datedateThe day being described — UNIQUE
food_scoresmallint0=Good, 1=Some weakness, 2=Poor, 3=Really bad (nullable)
food_notestextOptional short notes about the day
created_attimestamptzAuto-set on insert

health_config

App-wide configuration key-value store.

ColumnTypeDescription
key PKtextConfig key (e.g. rolling_avg_days)
valuetextConfig value (cast in app)
labeltextDisplay name for Admin tab
descriptiontextHelp text shown in Admin tab

Seeded values: rolling_avg_days=14, frozen_target=235, goal=220, commit=0.25

leg_logs

Daily leg circumference measurements.

ColumnTypeDescription
id PKuuidPrimary key
log_datedateDate of measurement — UNIQUE
right_anklenumeric(4,2)Right ankle circumference in cm
left_anklenumeric(4,2)Left ankle circumference in cm
created_attimestamptzAuto-set on insert

leg_context

Prior day context for leg measurements — sock usage and activity score captured the morning after.

ColumnTypeDescription
id PKuuidPrimary key
context_datedateThe day being described — UNIQUE
wore_socksmallint0=No, 1=Yes — wore compression sock
activity_scoresmallint0=Good, 1=Mixed, 2=Difficult (nullable)
activity_notestextOptional short activity notes
created_attimestamptzAuto-set on insert

habit_categories

ColumnTypeDescription
id PKuuidPrimary key
nametextCategory name — UNIQUE
created_attimestamptzAuto-set on insert

Seeded: Family, Health, Professional, Personal

habits

ColumnTypeDescription
id PKuuidPrimary key
nametextHabit name
category_id FKuuidReferences habit_categories(id)
descriptiontextFree text — supports links and notes (nullable)
frequency_typetext'daily' or 'weekly'
weekly_targetsmallintTimes per week target (null for daily habits)
is_activebooleanfalse = archived
created_attimestamptzAuto-set on insert

habit_logs

ColumnTypeDescription
id PKuuidPrimary key
habit_id FKuuidReferences habits(id)
log_datedateDate of completion
created_attimestamptzAuto-set on insert

UNIQUE constraint on (habit_id, log_date) — one completion per habit per day.

Deployment Workflow

  1. Download updated file(s) from Claude
  2. Go to github.com/danielgomez909/decent-health
  3. Click the file → pencil icon to edit, or drag-and-drop to upload
  4. Add a short commit message → click Commit changes
  5. Vercel auto-deploys in ~30 seconds
  6. Verify at the Vercel URL
  7. For icon updates: remove app from home screen and re-add via Safari → Share → Add to Home Screen
